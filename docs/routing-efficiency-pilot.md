# Routing efficiency and quality pilot

`fixtures/routing-pilot-v1/pilot.json` freezes three deterministic, file-oracle tasks. Each execution must materialize a fresh workspace per task/arm; prompts, seeds, permissions, and oracles are identical. Only delegation/routing changes across `single-sol-medium`, `all-sol-multi-agent`, and `routed-sol-terra`.

Use `scripts/extract-routing-pilot-telemetry.mjs` on raw JSONL session records plus the run's stdout and stderr. It retains actual model/effort, the host-provided input/cached/output/reasoning token fields when present, tool calls, persisted-timestamp elapsed time, retries, rework, and quality outcomes. It also reports structured operational evidence for sampling retries, rejected spawn attempts, command rejections, unknown-agent/fallback/thread-limit signals, and command/collaboration activity. Missing fields are `null` with `unavailable` availability. Currency is always unavailable unless separately supported by billable evidence; this pilot never converts tokens to dollars. Every reusable run stores a contract fingerprint and invocation receipt (argv, prompt hash, permissions, and start-state fingerprint); a changed task seed, prompt, oracle, arm, or invocation contract invalidates the prior run. All three task seeds are intentionally unsatisfied, including the semantic-role repair; the native-provenance task requires a produced `native-report.txt`.
